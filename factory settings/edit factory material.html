Edit Factory Material

Similar to the CAM page mapping, you can edit material aliases and bend materials (materials used in Flux Bend). The material editor dialog can be viewed using the command factory → sheets → map → materials to map materials to the alternate names.

The various mapping functionalities are described below:

  • Assign a bend material to the selected material or raw material.

  • Edit the alternate name (aliases) of a material.

  • Edit the alternate names of raw materials.

Bend-material

The materials in JFY+ and Flux might not always match directly, and you need to map the materials correctly for the tooling process in the Flux Engine. JFY+ assigns the mapped bend material to the part before sending it to the Flux Engine for bend/cut CAM. The bend material combo displayed in this UI displays all materials defined in Flux. Here’s how you can approach this:

Material → Bend-material mapping

The material assigned in JFY+ should be selected first. You’ll need to identify the material being used for the part from the options on the left. This represents the material as defined in JFY+. The combo box on the right displays all available materials defined in Flux. You must select a material from this list that is appropriate for the material assigned in JFY+.

  • Aluminum (which is the material defined in JFY+) is mapped to AlMg3 (which is the material defined in Flux).

  • When the part is prepared for tooling, the AlMg3 material is what will actually be used by the Flux Engine during the tooling operation.

Raw-material → Bend-material mapping

JFY+ material mapping can now assign bend materials directly to raw-materials. By default, a raw material (which is a combination of material and thickness) will inherit the bend material from the parent material type, but you also have the flexibility to override this inherited bend material for specific thickness(es).

  • When a raw-material (material + thickness) is defined, it will inherit the bend material from the parent material type.

  • This mapping will overrride the bend material for a specific raw-material. Assign the material inherited bend-material to reset.

  • Aluminum-25 (which is the raw-material defined in JFY+) is mapped to 1.4301 (which is the material defined in flux).

  • The part which uses this raw material will use this assigned bend material and not the material level mapping during the Flux engine tooling operation.

Material Aliases

Enter an alias name of the material, one-per-line, in the Aliases textbox. When importing new parts, these aliases are used to map the external material names like ERP, CAD systems etc. to the one defined in JFY+. This is a many-to-one map where multiple aliases can be assigned to a single JFY+ material. While an alias name must be used just once. The UI enforces this rule if the alias is used again for some other material.

Raw Material Aliases

Like material-aliases, the raw-material aliases allow to provide alternate names for material + thicknesses. These short names later can be used in a spreadsheet to provide a single entry for both material + thickness.
