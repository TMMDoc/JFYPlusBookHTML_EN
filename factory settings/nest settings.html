Nest Settings

Nest Settings

These settings are used to control the auto-nest behavior in JFY+.

Mix Parts from Multiple Jobs

When enabled, parts from multiple jobs are intermixed on a single layout. When unchecked, jobs are nested in a single-job-at-time standalone mode.

Interactive sheet-name prefix

JFY+ uses English letter P as a prefix in the layout names it generates. The name is in format: <Prefix><Running-Layout-Number>.<Extension>. For example, a MetaCAM layout is named as P10.lyt, P11.lyt etc. If this value is not set, the default prefix P is used both for the interactive as well as the automatic nested layouts. When set, the prefix specified here is used for interactive nests whereas the default prefix is used for automatic nests.

Update sheet inventory…​ on nest (un)release

Enable switch Update sheet inventory…​ on factory • settings • job page to update the sheet inventory when a nest is released to (or recalled from) a machine for production. JFY+ displays Insufficient Sheets report when the available sheets fall short of the required to complete the production.

  • The required sheet count is pulled from the inventory when nests are released for production.

  • The sheets are added back to the inventory when the nest is recalled.

Automate remnant sheet inventory management:

In JFY+, remnants were automatically utilized and given priority over standard sheets during processing. This behavior was consistent and applied universally without the option to modify or disable it.

Using remnants automatically was helpful in many cases, there were situations where more control was needed over when and how remnants were used. This created the need for an option to turn the automatic priority of remnants on or off based on specific preferences.

When the switch is enabled, the software automatically uses remnants without manual input. If the switch is disabled, remnant management becomes interactive requiring user decisions.

Automate twin-line nesting

Select this option on factorty → settings → jobs to enable twin-line automation.

The twin-line feasibility is evaluated during the Part auto tooling when this option is selected. The part field Can Twinline is set reflecting the feasibility result.

Can Twinline search in the library page can be used to filter parts that are twin-line feasible.

The Twinlined parts selection filter option can be used in creating a new jobs and assemblies with parts that can common-line. This option is available only for twin-line feasible parts.

Do not auto-release layouts

Select this option under factory → settings → jobs to prevent the auto-release of layouts after auto-nesting.

JFY+ auto-releases layouts that satisfy the re-nesting packing cut-off condition. When this option is enabled, it will prevent the auto-release of layouts, and the packing cut-off will be used to gather the repacking candidates as before.
